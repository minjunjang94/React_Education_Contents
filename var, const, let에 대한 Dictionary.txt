var 의 문제
1. 함수 스코프 // 함수를 벗어나서 변수를 사용하면 에러 발생
-> 함수가 아닌 프로그램의 가장 바깥에 정의하면 전역변수가 된다. 
-> var 키워드 없이 변수를 정의하면 전역 변수가 된다.
   @이런 상황에서 명시적 에러가 발생하도록 하려면 파일 상단에 use strict를 선언해야한다.
-> var는 함수 스코프이기 때문에 for과 같은 반복문에서 정의된 변수가 반복문이 끝난 이후에도 계속 남는 문제가 있다.

2. 호이스팅 // var로 정의된 변수는 그 변수가 속한 스코프의 최상단으로 끌어올려진다.
-> 변수 MyVar을 정의하지 않고 console.log(MyVar)를 잡으면 에러를 발생하지만,
    console.log(MyVar)을 사용한 뒤에 Var MyVar = 1; 를 정의하면 먼저 발생하는 로그엔 에러가 발생하지 않는다.
    undefined가 출력되는 문제점이 발생한다.


const, let 은 블록 스코프이다. => 블록을 벗어나면 변수를 사용할 수 없다.
const와 let으로 정의된 변수도 호이스팅된다. 하지만 변수를 정의하기 전에 그 변수를 사용하려하면 참조에러 발생.

const는 밴수를 재할당 불가능하게 만든다. (let은 가능)

const bar = 'a';
bar = 'b'; //에러발생
let foo = 'a';
foo = 'b'; //정상
